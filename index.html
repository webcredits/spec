Webcredits is a a way to model credits and payments between agents on the world wide web, using Linked Data.

This document in an informal draft published at the w3c web payments community group

Introduction

The vast majority of money in the world exists in the form of Credit. Yet there is no way to easily model a credit between counterparites on the web using the Linked Data Cloud. This specification proposes a vocabulary and examples that model credits between two agents, which can also be used to send credits to each other, building up a ledger of balances. In this sense, a modular kernel of an economy can be created while delegating other functions such as, authentication, signing and identity to other specifications.

Web Payments Community Group

Webcredits is one of several related specifications being discussed by the Web payments Community Group. Implementers interested in alternative approaches and complimentary protocols should join this group to learn more.

Ontology

The ontology / vocabulary is at https://w3id.org/cc. The prefix cc will be used in examples.

Ledgers

A ledger is a two column list that contains a URI and a balance, or amount. Each Ledger will generally be linked to a wallet that contains the default currency. 
      URI cc : amount 1.23 .

The Amount indicates the balance of the agent.

Credits

      <#this> a cc : Credit;
        cc : source URI ;
        cc : destination URI ;
        cc : amount URI .

The Source indicates the source of the credit.

The Destination indicates the destination of the credit.

The Amount indicates the amount of the credit.

Wallets

A wallet is a container of a ledger, it is the authority by which the ledger is defined. It also will show how to find balances, and change the state of the ledger. 
      <#this> a Wallet ;
      cc : api URI .

The API shows where to get balances and add new credits.
